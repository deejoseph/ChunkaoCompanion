## 一、项目整体架构

```
ChunkaoCompanion/
├── 用户系统
├── 日历与进度看板
├── 知识点学习（AI助教 + 图解）
├── 刷题系统（真题卷 + AI生成卷）
├── 数据分析（热点图 + 趋势图 + 个人诊断）
└── 家长反馈（邮件推送）
```

---

## 二、各模块详细设计

### 1. 注册环节

**字段设计：**
- 学员姓名（用于称呼展示）
- 家长邮箱（用于每日/每周进度邮件）
- 春考日期（默认1月第一个周末，允许修改）
- 选科（大三门固定，可后续扩展）

**技术注意：**
- 邮箱需验证（发送确认邮件，防止填错）
- 春考日期用于倒计时和进度规划

---

### 2. 日历与进度看板

**核心展示：**
- 顶部大数字：距离春考还有 **XXX** 天
- 日历视图：标记已学习日期、待复习日期
- 进度条：语/数/英各科完成百分比
- 今日任务卡片：根据进度和弱点智能推荐（1个知识点 + 3~5道题）

**技术点：**
- 日历组件可选用 FullCalendar 或手写简单月历
- 进度存储：每个知识点标记“未开始/学习中/已掌握”

---

### 3. 知识点学习

**知识点结构（以语文为例）：**

| 学科 | 一级分类 | 二级知识点 |
|-----|---------|-----------|
| 语文 | 文言文 | 实词、虚词、句式、翻译 |
| 语文 | 现代文阅读 | 概括题、赏析题、作用题、含义题 |
| 语文 | 作文 | 审题、立意、结构、素材 |
| 数学 | 代数 | 函数、数列、不等式 |
| 数学 | 几何 | 立体几何、解析几何 |
| 数学 | 概率统计 | 概率、统计 |
| 英语 | 词汇语法 | 高频词汇、时态、从句 |
| 英语 | 完形阅读 | 完形技巧、阅读定位 |
| 英语 | 作文 | 应用文、议论文模板 |

**知识点页面内容：**
1. **资料整理的核心内容**（你买的资料里提取的要点）
2. **AI助教入口**（对话框形式，可追问）
3. **图解区域**（预生成或实时生成）

**AI助教提示词设计（重要）：**
```
你是一个春考[数学/语文/英语]助教。学生正在学习[知识点名称]。
请根据以下要求回答：
1. 用通俗易懂的语言解释这个知识点
2. 数学知识点必须包含推导过程，步骤清晰
3. 如需要图解，请用文字描述图解内容（系统会调用生成图片）
4. 最后给出1个典型例题和解题思路
5. 控制在300字以内
```

**图解生成：**
- 提前用 DALL-E/Midjourney 生成常见图解（函数图像、几何图形、语法结构图等）
- 或接入实时生成API（如混元文生图）

### 4. 刷题系统

#### 模块A：真题/模拟卷库

**数据结构：**
```json
{
  "paperId": "2024_chunkao_math_01",
  "year": 2024,
  "type": "真题卷",
  "subject": "数学",
  "questions": [
    {
      "qId": "M-2024-01",
      "type": "选择题",
      "difficulty": 0.7,
      "knowledgePoint": "函数单调性",
      "content": "...",
      "answer": "B",
      "analysis": "..."
    }
  ]
}
```

**功能：**
- 按年份/学科筛选
- 限时模拟（计时器）
- 自动批改（选择题/填空题）+ 主观题参考答案比对

#### 模块B：AI生成试卷

**核心逻辑：**
1. 题目分类体系（定义好规则）：

| 学科 | 题型编号规则 | 难度分级 |
|-----|-------------|---------|
| 数学 | M-选择-01~20, M-填空-01~10, M-解答-01~8 | 1~10级 |
| 语文 | C-文言-01~30, C-现代-01~20, C-作文-01~15 | 1~10级 |
| 英语 | E-完形-01~15, E-阅读-01~25, E-作文-01~10 | 1~10级 |

2. **AI生成约束提示词：**
```
请生成一道[数学] [选择题]，知识点为[函数单调性]，难度等级[5/10]。
要求：
- 题干风格模仿春考真题
- 选项4个，只有一个正确
- 输出格式严格遵循以下JSON：
{
  "type": "选择题",
  "difficulty": 5,
  "knowledgePoint": "函数单调性",
  "content": "题干内容",
  "options": ["A...", "B...", "C...", "D..."],
  "answer": "B",
  "analysis": "解析..."
}
```

3. **生成策略：**
   - 根据学生的错题分布，优先出薄弱知识点的题
   - 保持整卷题型/难度比例与真题一致（可参考历年真题统计）
   - 每次生成后人工抽检（初期）→ 逐步自动化

### 5. 数据分析

**三个维度：**

| 图表类型 | 数据来源 | 作用 |
|---------|---------|------|
| 知识点热点图 | 历年真题知识点出现次数统计 | 看清高频考点，优先学习 |
| 出现趋势线性图 | 近5~10年各知识点出现频次变化 | 预测今年可能考什么 |
| 个人准确率雷达图 | 学生练习数据 | 定位薄弱点，推荐复习 |

**技术实现：**
- 图表库用 ECharts 或 Chart.js
- 热点图用色块矩阵展示
- 趋势线用折线图 + 线性回归预测线

**提醒逻辑：**
```
if 知识点高频 and 学生准确率 < 60%:
    标记为“紧急关注”
    推送到今日任务卡片
```

### 6. 家长反馈

**邮件内容模板：**
```
【春考伴学】学员[姓名] [日期] 学习报告

今日完成：
✓ 学习了[知识点名称]（25分钟）
✓ 完成练习题[正确率：80%]
✓ 错题：[题目简述]

本周累计：
- 学习天数：5/7天
- 掌握知识点：12个
- 刷题数量：45道，平均正确率：72%

薄弱提醒：[知识点] 准确率偏低，建议加强

距离春考还有 XXX 天，继续加油！
```

**技术实现：**
- 使用 Nodemailer 或云函数邮件服务
- 可设置每日/每周发送频率
- 支持家长手动触发“立即发送”

## 三、开发阶段建议

| 阶段 | 时间 | 核心任务 |
|-----|------|---------|
| Phase 1 | 第1-2周 | 用户注册 + 日历看板 + 基础知识库录入 |
| Phase 2 | 第3-4周 | 知识点学习页 + AI助教接入（先做语文/数学） |
| Phase 3 | 第5-6周 | 真题卷录入 + 刷题模块 + 自动批改 |
| Phase 4 | 第7-8周 | AI生成试卷（先做规则生成，再接入LLM） |
| Phase 5 | 第9-10周 | 数据分析图表 + 诊断推荐 |
| Phase 6 | 第11-12周 | 家长邮件 + 联调测试 + 内容填充 |
