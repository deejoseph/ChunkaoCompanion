### 💭 关于出题机制的技术思考

你提的问题很好，我帮你分析两种方案的优劣：

| 方案 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| **AI生成题目** | 题目无限、个性化强、可针对薄弱点 | 题目质量不稳定、可能超纲、格式不规范 | 日常练习、拓展训练 |
| **随机抽取题库** | 题目质量有保证、贴近真题、格式规范 | 题量有限、可能重复 | 模拟考试、真题训练 |
| **混合方案** | 结合两者优点 | 实现复杂 | **推荐！** |

**推荐方案：**
```
基础题库（真题+模拟卷）→ 随机抽取 → 保证质量
    ↓
薄弱知识点 → AI生成补充题 → 个性化训练
    ↓
新题加入题库 → 持续扩充
```

经过多轮测试（解三角形、函数集合、代数证明），得出以下结论：

| 模式 | 模型 | 平均耗时 | 适用场景 | 准确率 |

| 🚀 中速模式 | qwen2-math:7b | 20-30秒 | 疑难题目、代数证明、复杂计算 | 高 |

## 详细设计

### 1. AI 提取答案库

在批量处理 Word 文档时，同时从教师版 PDF 中提取：

```
专题01 教师版.pdf
├── 题目1: 计算 ∫₀¹ x² dx
├── 答案1: 1/3
├── 解析1: 使用幂函数积分公式...
├── 题目2: ...
└── 答案2: ...
```

存储格式：

```json
{
  "topicId": "chinese_专题01",
  "questions": [
    {
      "id": 1,
      "type": "calculation",
      "question": "计算 ∫₀¹ x² dx",
      "answer": "1/3",
      "analysis": "使用幂函数积分公式..."
    }
  ]
}
```

### 2. 生成答题卡

根据题目生成可打印的答题卡（HTML/PDF）：

```
┌─────────────────────────────────────────┐
│  专题01 名篇名句默写 - 答题卡              │
├─────────────────────────────────────────┤
│  1. ______________                      │
│  2. ______________                      │
│  3. ______________                      │
│  4. ______________                      │
│  5. ______________                      │
├─────────────────────────────────────────┤
│  条形码/二维码区域                        │
└─────────────────────────────────────────┘
```

### 3. 拍照批改流程

1. 学生打印答题卡并手写答案
2. 拍照上传
3. 系统识别答案区域
4. 与答案库比对
5. 生成评分报告
